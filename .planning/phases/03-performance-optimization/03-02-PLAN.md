4. Modify `getQueueStats()` to use cache:
   - At the beginning of the method (before `const matchStage`), add cache lookup:
     ```typescript
     const ttl = this.ctx.options.statsCacheTtlMs;
     const cacheKey = filter?.name ?? '';

     if (ttl > 0) {
       const cached = this.statsCache.get(cacheKey);
       if (cached && cached.expiresAt > Date.now()) {
         return cached.data;
       }
     }
     ```
   - After the `return stats;` line at the end of the try block (before returning), replace with cache-then-return:
     ```typescript
     // Cache the result if TTL is enabled
     if (ttl > 0) {
       // LRU eviction: if cache is full, delete the oldest entry
       if (this.statsCache.size >= JobQueryService.MAX_CACHE_SIZE) {
         const oldestKey = this.statsCache.keys().next().value;
         if (oldestKey !== undefined) {
           this.statsCache.delete(oldestKey);
         }
       }
       // Delete existing entry first so re-insertion moves it to end (Map insertion order = LRU)
       this.statsCache.delete(cacheKey);
       this.statsCache.set(cacheKey, {
         data: stats,
         expiresAt: Date.now() + ttl,
       });
     }

     return stats;
     ```
   Note: Map preserves insertion order, so newly accessed/inserted entries go to the end. On eviction, `keys().next().value` gives the least recently inserted (oldest) entry. This is a simple LRU approximation — entries that are re-queried get re-inserted at the end.
