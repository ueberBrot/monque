**Test: "should handle concurrent cleanup from two instances without data corruption"**

Follow the exact pattern of the existing retention tests. Key design per CONTEXT.md locked decisions:

1. **Setup:** Create a unique collection name. Configure TWO Monque instances sharing the same `db` and `collectionName`, each with short cleanup intervals:
   ```ts
   const monque1 = new Monque(db, {
     collectionName,
     pollInterval: 1000,
     jobRetention: {
       completed: 5000,
       failed: 5000,
       interval: 100,
     },
   });
   const monque2 = new Monque(db, {
     collectionName,
     pollInterval: 1000,
     jobRetention: {
       completed: 5000,
       failed: 5000,
       interval: 100,
     },
   });
   ```
   Push both to `monqueInstances` array for cleanup in `afterEach`.

2. **Seed data:** Insert a mix of documents directly via `collection.insertOne()` using `JobFactoryHelpers`:
   - 3 old completed jobs (`updatedAt` = 6s ago) — should be deleted
   - 3 old failed jobs (`updatedAt` = 6s ago) — should be deleted
   - 2 recent completed jobs (`updatedAt` = 100ms ago) — should survive
   - 2 recent failed jobs (`updatedAt` = 100ms ago) — should survive

   Give each a unique `name` field (e.g., `'old-completed-1'`, `'old-failed-2'`, `'recent-completed-1'`, etc.) for verification.

3. **Execute:** Initialize and start both instances:
   ```ts
   await monque1.initialize();
   await monque2.initialize();
   monque1.start();
   monque2.start();
   ```

4. **Wait for cleanup:** Use `waitFor()` to poll until ALL old jobs are gone:
   ```ts
   await waitFor(
     async () => {
       const oldCount = await collection.countDocuments({
         name: { $regex: /^old-/ },
       });
       return oldCount === 0;
     },
     { timeout: 5000, interval: 50 },
   );
   ```

5. **Assert:**
   - All 6 old jobs deleted: `countDocuments({ name: { $regex: /^old-/ } })` === 0
   - All 4 recent jobs survive: `countDocuments({ name: { $regex: /^recent-/ } })` === 4
   - Total remaining === 4 (no phantom documents created)
   - Verify specific recent jobs exist by name to ensure no wrong documents were deleted
