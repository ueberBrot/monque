  <action>
    **1. Verify explicit return type** on `packages/core/src/jobs/document-to-persisted-job.ts`:

    The goal: if someone adds a new required field to `Job<T>` but forgets to add it to the mapper, TypeScript compilation fails.

    Approach: The function already has an explicit `PersistedJob<T>` return type annotation. The object literal assigned to `const job: PersistedJob<T> = { ... }` must satisfy all required fields of `PersistedJob<T>` — omitting one causes a compile error. This is the simplest and most idiomatic TypeScript approach: zero extra type machinery, and the return type annotation is the exhaustiveness guard itself.

    For optional fields (lockedAt, claimedBy, etc.), these are conditionally set after the initial object literal. Since they're optional on `PersistedJob<T>`, the compiler won't catch missing optional fields — but round-trip tests provide runtime coverage for this case.

    No additional type-level code is needed. The function's return type does all the work.

    **2. Write unit tests** in `packages/core/tests/unit/document-to-persisted-job.test.ts`:

      - Call mapper, assert optional fields are NOT present (not even undefined — use `'lockedAt' in result` check)
    - These round-trip tests catch field-dropping regressions at runtime (complementing the compile-time return type check)
